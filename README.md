**iFolks Demo** is the official reference full-stack application of the **iFolks** ecosystem. It is entirely generated and maintained by the [iFolks Generator](https://github.com/ifolks-open-source-community/ifolks-generator) based on the domain model defined in `data-model/skeleton.xml`.

This repository serves as a showcase and blueprint of the standard multi-tier architectural standards generated by the tool, spanning from a Spring Boot backend to an Angular frontend.

---

## 🏗️ 12-Tier Architecture Structure

The application is structured into specialized Maven sub-modules and an Angular client, ensuring maximum separation of concerns and high maintainability:

| Module | Tier | Description |
| :--- | :--- | :--- |
| **`demo-api`** | Interface | Defines the domain interfaces, filters, sortings, view DTOs, and base services. |
| **`demo-model`** | Domain Model | JPA Entities mapped to the database (Region, Country, City, Calendar, Fool, Stupid). |
| **`demo-persistence`** | Data Access | Spring Data JPA Repositories and specifications for complex criteria-based searches. |
| **`demo-components`** | Business Logic | Maps, validators, state managers, and business logic coordinators. Uses Jackson `JsonMapper`. |
| **`demo-services`** | Application Service | Transaction-managed implementation of the base services. |
| **`demo-rest`** | Web Layer | Spring MVC Rest Controllers exposing endpoints, handling exceptions, and managing JSON serialization. |
| **`demo-populator`** | Database Seeding | Populates the local database (HSQLDB / PostgreSQL) with starter and reference data read from CSV files. |
| **`demo-tests`** | Integration Tests | Unit and integration test suite executing under **JUnit 5 Jupiter** and `@SpringJUnitConfig`. |
| **`demo-ui`** | Angular Frontend | Complete single-page application client featuring responsive tables, details pages, search modals, and routing. |

## 🛠️ Getting Started

### 1. Build and Run the Backend
Ensure you have **Java 21** and **Maven 3.9+** installed:
```bash
# Compile and package the entire multi-module application
mvn clean install
```
This will also trigger the JPA populator to seed database reference files and run all JUnit 5 integration tests successfully.

### 2. Run the Angular UI
Ensure you have **Node.js** installed, then launch the dev server:
```bash
cd demo-ui
npm install
npm run start
```
Open your browser at `http://localhost:4200` to interact with the generated administrative lists, detail forms, search modals, and references.

## 📝 Domain XML Model (`skeleton.xml`)

All layers in this demo (database schemas, java classes, rest endpoints, typescript models, and angular components) are automatically synchronized and rebuilt from `data-model/skeleton.xml`. 

Whenever the domain requirements evolve (e.g. adding a new field, changing selection behaviors, or adding a new table), running `igen generate` instantly regenerates the entire multi-tier codebase while preserving architectural integrity.
